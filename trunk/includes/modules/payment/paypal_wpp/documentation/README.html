Introduction

This payment module works with PayPal's Website Payments Pro credit card gateway and is FULLY COMPLIANT with their rules. This was developed on a stock 2.2ms2 install.

Please note: Due to Paypal's rule about express checkout being integrated with direct payment, this contribution is not trivial to install. I've made this as simple to install as possible, but it requires a minimal level of PHP scripting knowledge.

This package will modify your store in the following ways
-If a logged in customer clicks on "Checkout," they're taken to the shipping page with a "Paypal Express Checkout" button at the top. If they click on the "Express Checkout" button, they'll be taken to Paypal where they'll choose a shipping address and payment method, then be sent back to the checkout shipping page to choose a shipping method. From there, depending on your settings, it'll either go to checkout payment hiding the other payment options but displaying other modules, like CCGV, or to checkout confirmation to complete the order.

-If a visitor without an account, or someone who isn't logged in, clicks on "Checkout," they'll be taken to the login page where there are now three boxes. They can either use express pay, create an account, or login. If they use express pay, they'll be taken to Paypal where they'll choose a shipping address, then be sent back following the process above. If they're not a current customer, based on the information returned from Paypal, an account will automatically be created for them and the account details emailed to their email address (notification is optional). If they're an existing customer, they'll be logged in and their order will be added to their account.

-If they log in, or create an account, and go through the normal checkout process, then they can type in their credit card numbers and the direct payment API will charge their card transparently.

!!WARNING!! IF YOU REMOVE THE EXPRESS CHECKOUT BUTTON FROM YOUR CART, YOU ARE VIOLATING PAYPAL'S TOS AND THEY HAVE THE RIGHT TO DISALLOW ACCESS TO DIRECT PAYMENT AND/OR CLOSE YOUR ACCOUNT. THIS CONTRIBUTION, AS DOWNLOADED, IS FULLY COMPLIANT WITH THEIR RULES.
Requirements

To use this, you need these things (ask your webhost if you don't know):
- OSCommerce 2.2ms2
- PHP 4.3.0+
- cURL
- An SSL certificate installed
- You need a business account with Paypal
- You must be approved for Website Payments Pro direct transactions
(there's a request form on Paypal's site on your "Overview" page when you login)
- The API certificate that you receive from PayPal after being approved for WPP
Pear Modules

I've uploaded the pear modules you'll need in two separate packages in case you don't want, or can't, install them on your server using pear.

Pear Modules - Part 1
Pear Modules - Part 2

For full disclosure, you should know that I have modified one file because the payment module would timeout otherwise. In the file:
/catalog/pear/Services/PayPal/SOAP/Transport/HTTP.php

I've modifed line 62 from:

var $timeout = 4;

to

var $timeout = 300;

If you have a healthy paranoia and don't want to install the pear modules I've uploaded, you need to use pear to install the following packages (if they're not already installed): Net_URL Net_Socket HTTP_Request Log

You'll also need to download PayPal's Website Payments Pro 2.0 SDK and install the pear module contained within. I suggest you change that timeout variable mentioned above, though. A complex set of API calls over an encrypted connection needs more than 4 seconds to complete. You should set this to at least 120.
Installation Instructions

0. BACKUP YOUR DATABASE AND ALL OF YOUR FILES!

1. You need an API certificate, API username, and API password. Assuming you've already requested and been approved for Website Payments Pro direct payments, do this:

    a. Goto My Account -> Profile.
    b. Click the API Access link.
    c. Click the API Certificate Request link.
    d. Complete the certificate request.
    e. Click on download and save the file to your computer.


2. Copy the pear modules that you downloaded above into your /catalog/ directory so that they are located in /catalog/pear/


3. Copy all of the files and directories from this zip file to your catalog directory keeping true to the directory structure. For the files that already exist, you will need to either overwrite them, or manually update them by opening each one and copying the code between these tags:

//---PayPal WPP Modification START ---//
//---PayPal WPP Modification END ---//


4. Import the included SQL file called "paypal_wpp.sql" into your database. (Using phpMyAdmin)


5. Upload the API certificate you downloaded from Paypal to: /catalog/includes/modules/payment/wpp_cert/

For security, I have included a .htaccess file that denies access to any file put in there (DO NOT FORGET TO UPLOAD IT), so don't worry about anyone downloading it. As of 0.3, you can store your certificate anywhere you want. Simply type in the full path in the module's admin panel.


6. Go into your Administration section -> Modules -> Payment and enable the "PayPal Direct Payment & Express Checkout" module.


7. Fill in your API username, password, and the filename of your API certificate that you just uploaded and change the other settings.


8. Click "Update" and you're done. Assuming everything's installed properly, and the planets are aligned correctly, it should start processing your cards automagically.
Updating Instructions

Follow these instructions to the letter or you might leave old values in your database!

1. Go into your administrative section -> Modules -> Payment and copy down all of the settings for this module.

2. Click on "Remove" and uninstall this module

3. Refer to the changelog at the bottom of this document and update the files listed as changed

4. Go into your administrative section -> Modules -> Payment and install the PayPal Direct Payments & Express Checkout module updating the settings with what you wrote down in step 1.

Troubleshooting

Q: When the visitor first clicks PayPal Express, it never goes to PayPal

OR

Q: When the visitor clicks on the PayPal Express button or tries to checkout using Direct Payment, it goes to a blank page.
A: Either your API username or password are wrong, your certificate is FUBAR'd, or you didn't install the pear modules correctly. You can also try increasing the timeout in pear/Services/PayPal/SOAP/Transport/HTTP.php

Change this line:
var $timeout = 4;

to

var $timeout = 300;


Q: When the customer returns from PayPal after selecting "Express Checkout", only their name is returned.
A: PayPal will not return a customer's personal information to a non-https URL. Fix your OSC configuration and install an SSL certificate if necessary, then try again.

Q: It's declining all of my transactions and giving a 105xx error!
A: Only PayPal can help you there. They're declining the cards probably because you don't have Direct Payment completely setup under your account. Contact PayPal for assistance with this error.

Q: It's declining transactions and giving a 10417 error!
A: This means that the customer doesn't have enough money in their bank or on their credit card to pay for this purchase. Most of the time, Paypal is incorrect so you need to contact PayPal and ask them about it.

Q: PayPal's only accepting ~60% of all credit card transactions. Is this a bug in your module?
A: No, this this module submits information as the customer has entered it and their card getting declined is a combination of bad personal information and PayPal's servers. It has been noted that PayPal has a low acceptance rate even when the information is correct, but YMMV.

Q: Why do my international customers' cards get declined?
A: PayPal is a US-only processor. While I've made changes to the module so that it will accept some international cards, it's not perfect. If you expect to have many international (non-US) customers, I suggest you go with a traditional merchant account/gateway.

Q: When the customer returns from PayPal after selecting "Express Checkout", only their name is returned.
A: PayPal will not return a customer's personal information to a non-https URL. Fix your OSC configuration and install an SSL certificate if necessary, then try again.

Q: I get a "(2030) Unable to find API certificate file" error!
A: Make sure that you:
    A) Download the certificate you receive from Paypal and put it inside of the
        /catalog/includes/modules/payment/wpp_cert/ directory
    B) Ensure the path set to that directory is correct in the module's admin settings

Q: The ec_process.php/checkout_confirmation.php/whatever.php comes up blank!
A: 99% of the time either your certificate is not installed with the module set to it or all your pear modules aren't installed. Check your error_log file for details.
