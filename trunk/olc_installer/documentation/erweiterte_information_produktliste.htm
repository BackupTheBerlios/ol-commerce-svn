Produkt-Listen

Standardmäßig werden alle(!) Produkt-Listen mit demselben Template "module/product_listing/product_listing_v1.html" erstellt,
(und auch mit demselben Programm-Code!) so dass eine systemweite Konsistenz aller(!) Produkt-Listen gewährleistet ist.

OL-Commerce v4/AJAX stellt eine Menge mehr Informationen in der Produkt-Liste bereit,
als dies z.B. in OL-Commerce v1.2 oder xtCommerce der Fall ist.

Zum einen dient dies zusätzlicher Information (*), zum anderen sind diese
Informationen notwendig, um den Shop abmahnungssicherer (*) zu machen.

Wenn man nicht die Standard OL-Commerce "product_listing_v1.html" verwenden kann/will,
dann sollte unbedingt diese Information in ein bestehendes Formular eingearbeitet werden!

Folgende "Smarty"-Variable sind im Template zusätzlich vorhanden:

PRODUCTS_MODEL * Artikel-Nummer
PRODUCTS_VPE * Verpackungseinheit (z.B.: "Gewicht: 100 gr")
PRODUCTS_MIN_ORDER_QTY * * Mindestbestellmenge
z.B.:
Mindestabnahme: 
 5 Stück (7,50 EUR)
SHOW_NEW_MARKER * Damit kann eine "Neu"-Grafik für neue Produkte eingeblendet werden 
{if $module_data.SHOW_NEW_MARKER}
<BR><img src="{$tpl_path}buttons/{$language}/new.gif" border="0">
{/if}
PRODUCTS_DATE_AVAILABLE* Produktverfügbarkeits-Datum
PRODUCTS_PRICE_RAW Interner unformatierter Preis (für Berechnungen im Template)
PRODUCTS_PRICE Anzeigepreis (formatierter Preis)
PRODUCTS_BASEPRICE * * Grundpreis (z.B.: "100 gr: 5,-- EUR")
PRODUCTS_SPECIALPRICE * Sonderpreis
PICTURE_DISCLAIMER * * Bild-Disclaimer (z.B.: "Abb. ähnlich")
PRICE_DISCLAIMER * Preis-Disclaimer

Beispiele:
Gemäß §19 UStG wird keine MwSt. erhoben.
Preis zzgl. Versandkosten (ab 3,– EUR)
(Für genauere Information anklicken!).

(Für einen "Kleinunternehmer"-Shop)		 Preis inkl. X%  MwSt. und
zzgl. Versandkosten (ab 3,– EUR)
(Für genauere Information anklicken!)

("X" wird Produktabhängig angezeigt: voller/reduzierter MwSt.-Satz oder keine MwSt.)

Der Minimale Versandkosten-Betrag (hier z.B. 3,-- EUR) wird aus allen konfigurierten Versandarten automatisch ermittelt!

Der "Versandkosten"-Link führt auf die "Versandkosten"-Information des Shops, angezeigt in einem "Pop-Up"-Fenster.
 

OL-Commerce v4 hat folgende weitere Besonderheiten

PRODUCTS_PRICE_RAW ist der unformatierte Preis, er entspricht dem in der Produktseite eingegebenen Preis.

OL-Commerce v4/AJAX erlaub für den Preis Werte kleiner, gleich und größer als Null!

Diese Preis-Zuordnung dient dazu, ein Produkt zwar aus dem Verkauf zu nehmen, es aber trotzdem in der Produktliste zu belassen, mit einem evtl. Hinweis auf ein Verfügbarkeitsdatum!

Damit kann über den PRODUCTS_PRICE_RAW (Produktpreis) das Anzeige-Verhalten wie folgt weiter gesteuert werden:

PRODUCTS_PRICE_RAW  = 0 Es wird kein Preis und kein Bestell-Button angezeigt
PRODUCTS_PRICE_RAW  < 0 Produkt ist nicht verfügbar, wird aber angezeigt!

Es wird ein (absoluter) Preis und kein Bestell-Button angezeigt

Es erfolgt ein Hinweis, dass das Produkt aktuell nicht verfügbar ist.

Hat gleichzeitig die Variable PRODUCTS_DATE_AVAILABLE einen Inhalt, wird dieser Datumswert als das erwartete Verfügbarkeitsdatum angegeben.

Hier werden die Felder "Preis" und "Verfügbarkeitsdatum" aus der Produktdefinition ausgewertet. Ist das aktuelle Datum größer als das definierte Verfügbarkeitsdatum, dann wird das Produkt automatisch wieder bestellbar und normal angezeigt.

(Das muss nicht extra zurückgenommen werden.)
PRODUCTS_PRICE_RAW  > 0 Normale Anzeige

Dieses Verhalten kann wie folgt in ein Template eingebunden werden:
 
  
{if $module_data.PRODUCTS_PRICE_RAW > 0 && $module_data.PRODUCTS_DATE_AVAILABLE == ''}
<td valign="middle" align="right">
	{$module_data.PRODUCTS_BUTTON_BUY_NOW}
</td>
{elseif $module_data.PRODUCTS_PRICE_RAW <> 0}
<td valign="top">
	<br>
	<font color="#FF0000" size="1"><b>
	{if $module_data.PRODUCTS_PRICE_RAW < 0 }
	{#product_not_in_stock#}<br></b>
	{/if}
	{if $module_data.PRODUCTS_DATE_AVAILABLE}
	<font color="#008000">({$module_data.PRODUCTS_DATE_AVAILABLE})</font>
	{/if}
	</font>
</td>

{/if}
